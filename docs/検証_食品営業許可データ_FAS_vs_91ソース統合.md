# 検証メモ：食品営業許可オープンデータ 「FAS単一」 vs 「91ソース統合」

作成日: 2026-07-16

食品店POIの悉皆性（網羅性）検証に使う「食品営業許可オープンデータ」について、2つのデータセットの
出自・規模・網羅性の違いを調査した記録。結論として **両者はスコープが根本的に異なる別物** である。

---

## 1. 2つのデータセット比較

| 項目 | 自前取得（FAS単一） | 大橋さん `facilities-all.csv`（91ソース統合） |
|---|---|---|
| 出典 | **厚労省FAS全国ポータルのみ**（i2fas） | **90個の異なるオープンデータ出典**を横断名寄せ |
| 列 | 25列・日本語（FAS原本のまま） | 16列・英語（正規化スキーマ） |
| 時点 | 2026年6月末の単一スナップショット | 出典ごとにバラバラ（**古い版=2021等も混在**） |
| 行数 | **1,089,349** | **1,449,538**（≒140万） |
| ユニーク | 993,483 施設（行番号=fBiz…で集計） | ≈1,216,919（名称+住所+業態のdistinct、近似） |
| サイズ | 360MB（345 B/行, UTF-8 BOM, CRLF） | 563.8MB（=537.7 MiB, UTF-8 BOM, LF） |

### 差 +36万行の内訳（大橋データの `sources` 列を解析）
- 個別出典の数 = **90**、出典の組合せパターン = 108
- FAS単独出典の行 = 422,159
- **県・市独自にしか無い行（FAS非該当, sourcesにi2fas無し）= 30,562**
- **歴史的スナップショット由来（「令和3年3月末時点」「2021年改正以前」等）= 50,627 行**
- FAS参照行 = 1,418,976（総行の97.9%）← 私の当月FAS実数108.9万を **33万超過** → 名寄せが緩く重複計上の可能性
- distinct(名称+住所+業態)=121.7万 < 総144.9万 → **約23万行が実質重複**（本人談「正規化は途中」）

→ 大橋データは **「網羅性は高いがノイジー（時点混在・重複・名寄せ精度不明）」**。

## 2. FAS全国ポータル（i2fas）からの取得方法

- サイト: 厚労省「食品衛生申請等システム 食品衛生公開データベース」 https://i2fas.mhlw.go.jp/faspub/
- 富士通系Strutsのステートフルアプリだが、**実ダウンロードは単純なGET1本**で足りる:
  - `GET https://i2fas.mhlw.go.jp/faspub/page/opendatadownload.jsp?param=<5桁自治体コード>_food_business_all.csv`
  - ALB Cookie があればイベントチェック不要で直接CSV取得可
- 全自治体コード（157件）は `IO_S011001_base.js` の `setArg_A_N*` 関数群から抽出
- **配信は当月版のみ・過去版は再取得不可** → 時系列で追うなら月次アーカイブが必要（更新頻度＝月1回・前月末締め）
- 保存先: `data/faspub_food_business/`（157 CSV + `_manifest_code_name.tsv`）※gitignore対象

### データ内の注意
- 日付列に入力ミス（`3022/05/31` 等の未来日）が混入。時系列分析時は妥当範囲フィルタ必須。
- 1施設が営業の種類（業態）ごとに複数行に展開される（993K施設→1.089M行）。

## 3. 網羅性の観点（重要）

FAS全国オープンデータは **「電子申請され、かつ掲載に賛同した施設のみ」の部分集合**。
紙申請・非同意の施設は載らない。一方、都道府県・市区の独自オープンデータの多くは
**許可台帳そのもの（全許可施設）** を公開している。

COVERAGE PDF が定義する収集優先度（網羅性の高い順）:
> ①自治体自身が公開 → ②管轄保健所設置主体が公開 → **③i2fas（FAS＝賛同施設のみ）** → ④無し

すなわち **FASは最も手薄いフォールバック層**。網羅性が目的なら県・市データを主・FASを穴埋めに使う
（＝大橋さんの統合設計が正しい方向）。

### 数字の整理（157 と 1,741 の混同に注意）
- **1,741** = 全国市区町村数（母数、e-Stat 2024-10-01時点）
- **157** = 食品営業許可を出せる「保健所設置主体」数（都道府県47＋指定都市20＋中核市62＋その他政令市5＋特別区23）
  ＝ 自前取得の157ファイルと一致（出典: 厚労省「設置主体別保健所数」令和8年4月1日現在）
- カバレッジ = 1,727 / 1,741 市区町村（99%）

## 4. 統合元ソース（92 distinct URL / 全1,727市区町村マッピング）

COVERAGE PDF の埋め込みリンク注釈を PyMuPDF で全抽出（総リンク数1,730、うちデータ行1,727）。

- `data/slack/coverage_municipality_links.csv` — **全1,727市区町村 × リンクの完全対応表**
  （列: 都道府県, 市区町村, 管轄, 収集経路, ライセンス, ソースURL）
- `data/slack/source_links.csv` — **distinct 92ソースURL** ＋ 各URLがカバーする市区町村数

収集経路の内訳（市区町村ベース）: i2fas(FAS) 1,037 / 管轄主体 618 / 自身 72
URLスキーム別 distinct: https 89 / http 2 / **ckan: 1**（`ckan:c9bf82c1…`＝内部参照, 40市区町村ぶん・要解決）
1URLで多数市町村をカバー（例: 長野県URL=75市町村, 福島県=56, 愛知県=49）。
BODIK(`data.bodik.jp`)のデータセットページは実CSV取得に1ホップ必要。

## 5. 「網羅性最大化データ」の作り方（推奨方針・未実施）

大橋さん設計（各市区町村＝最も完全で新しい1ソース採用→名寄せ）を「鮮度更新＋重複除去」して仕上げる:

1. FAS層は自前の当月版（2026-06, 157ファイル）で置き換え（クリーン・最新・取得済み）
2. 県/市の91ソースを最新版で再取得（古い2021版を差し替え。BODIK24件はリソースURL解決が必要）
3. 許可番号 or 正規化（名称+住所）で名寄せ・重複除去
4. 各市区町村＝「自身 > 管轄主体 > FAS」の優先で1レコードに寄せる

### 未確定の要検証点
- **多ソース統合はFAS単独より実際どれだけ網羅性が増えるか**。大橋データ上は「県独自のみ(FAS非該当)」が
  30,562行(約2%)だが、名寄せ精度次第で上乗せは僅少とも大きいとも読める → 自前FAS当月版と名寄せして実測すべき。
