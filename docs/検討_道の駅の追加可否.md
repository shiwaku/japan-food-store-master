# 検討：食料品店マスターに「道の駅」を含めるか

**作成**: 2026-07-19
**ステータス**: 🟡 **現状では含めない**（将来的に追加を検討）
**関連**: `設計_食料品店マスター構築.md` / `データ定義と抽出方法_食品店POI.md` / `検証_カテゴリ別比較_Overture_vs_OSM.md`

---

## 0. 結論（現時点）

- **Phase1 マスターには道の駅を含めない**。
- ただし、農村・中山間部の食料品アクセス判定精度を上げる**補完レイヤーとして将来追加を検討する**価値が高い。
- 追加する場合は「**別カテゴリ（`roadside_station`）＋物販ありに限定＋既存 fresh_food と空間 dedup**」を条件とする。

## 1. 背景・現状

- 現マスターのカテゴリと件数（Phase1）: `convenience 54,793 / supermarket 30,638 / fresh_food 10,070 / drugstore 7,729`。
- 直売所は `fresh_food`（Overture `farmers_market`・OSM `amenity=marketplace` 等）として**一部は既に捕捉済み**。
- 一方、**道の駅そのものは Overture/OSM で rest_area・tourism 系タグになりがちで、「食料販売施設」としてはほぼ捕捉できていない**（現マスターの穴）。

## 2. 含める価値（メリット）

1. **立地が食料品アクセス困難地域と一致** — 道の駅は中山間・農村部に偏在し、車を持たない高齢者の食料調達点として実質的に機能（産直・直売所で生鮮野菜・米・弁当等）。診断アプリの目的地として**限界的価値が最も高い**層。
2. **POI が取りこぼす食料供給機能を埋める** — `farmers_market` として拾えているのは一部のみ。道の駅の物販機能は現マスターで抜けている。
3. **公式・悉皆・ライセンスがクリーンなソースがある** — 国土数値情報「道の駅」（国土交通省、`P35` 系、全国約 1,230 箇所、座標付き、国土数値情報利用約款で再配布可）。POI より正確・完全で、二層設計（位置は最良ソース／数量は統計で検証）にそのまま乗る。

## 3. 含めない理由・留意点（現時点で見送る根拠）

1. **農水省の厳密な食料品アクセス定義から外れる** — 定義は経済センサスの**食料品小売業**ベースで、道の駅直売所はそこに入らない。コア小売（コンビニ・スーパー）と混ぜると「厳密定義」の集計が濁る。
2. **物販の有無でフィルタが必要** — 道の駅にはトイレ・情報提供のみの施設もある。無条件に全件入れると非食料施設が混入。
3. **既存 fresh_food との二重計上リスク** — farmers_market として既に入っている直売所と重複する可能性があり、空間 dedup が前提。
4. **全国件数への寄与は小さい** — 農水省加重では一般小売/直売は低ウェイト（約 5%）。価値は「農村部での局所的な到達可能性の底上げ」であって、全国網羅率を動かすものではない。

## 4. 追加する場合の実装方針（将来）

1. **取得**: 国土数値情報「道の駅」（`P35` 系）をダウンロード。属性に「産地直売所の有無」フラグがあるか確認。
2. **フィルタ**: 物販・直売機能があるものに限定（属性が弱ければ「原則含める＋明らかな非物販を除外」）。
3. **dedup**: 既存 `fresh_food`（farmers_market 由来）と 100m 近接で名寄せ、二重計上を排除。
4. **付与**: `cat = 'roadside_station'` として分離追加。
5. **分析での扱い**: コア小売とは別レイヤーにし、**ON/OFF 切替可能**に。「厳密定義（道の駅なし）」と「実アクセス（道の駅込み）」の両方を出せるようにする。

## 5. 位置づけ（まとめ）

道の駅は「全国網羅率を上げるため」ではなく、「**食料品アクセス困難地域の判定精度を上げる補完レイヤー**」として検討する。Phase1 では見送り、コアマスター（POI 由来の小売）が固まった後に、国土数値情報を用いた補完として再評価する。
