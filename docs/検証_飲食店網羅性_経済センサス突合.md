# 検証：飲食店営業の網羅性（経済センサス突合）

> 対象: 大橋本家 `facilities-all_after.csv`（geocode修正後）の飲食店営業施設
> 正解データ: **令和3年 経済センサス‐活動調査**「産業(小分類)別全事業所数－全国、都道府県、市区町村」
> （e-Stat statsDataId=`0004005687`、中分類 `76 飲食店`、2021年）
> 作成日: 2026-07-18 ／ 関連: `検証_食品営業許可届出データ_網羅性と検索API適性.md`

## 結論

**144万件の飲食店営業データは、全国で均質に使えない。** 許可施設数÷センサス飲食店事業所数の比率が
**都道府県で 0.03〜4.36（約140倍のばらつき）**。県ごとに網羅度が桁違いで、件数の少なさが
「店が無い」のか「データが無い」のか区別できない。

## 数字の前提（比率は単純カバー率ではない）

全国：許可 飲食店営業施設 **703,666**（name+address dedup） vs センサス 飲食店 **499,193** ＝ **比率1.41**。
許可がセンサスを上回るのは、**「飲食店営業許可」がセンサス「飲食店（産業分類76）」より広い**ため：
- 旅館・ホテル、集団給食、製造業の試食カウンター、スーパーのイートイン等も飲食店営業許可を持つ。
- 廃業施設が混在し得る（i2fasの`_all`は廃業含む）。

→ よって比率は「許可の網羅率」そのものではなく、**比率のばらつき**で穴を読む。
本来なら広い定義ゆえ比率>1になるはずの県が**比率<0.5**なら、明確な欠落。

## 3つのパターン

1. **比率<0.5＝欠落県**（新潟・秋田・青森・愛知・埼玉・神奈川・茨城・福井・徳島）。広い定義でも足りない＝真の穴。
   - 原因A（修正可能）: 本家ソース定義URLの恒常404（名古屋・川崎・藤沢・横須賀・青森・秋田・福井・滋賀 等）。
   - 原因B（構造的）: 管轄主体がODを出さずi2fas依存（新潟・茨城・徳島）＝賛同施設のみで薄い。
   - 原因C（表記）: 業種名が空欄（宮城73%・島根90%）＝records はあるが飲食店営業と判別できない。
2. **比率≈1（東京・鳥取・広島）**：妥当な網羅。
3. **比率>2（長野4.36・富山4.06・熊本・奈良・鹿児島・京都）**：管轄主体（県）が**全許可台帳**を公開し、
   広義の飲食店営業許可（旅館・給食・廃業含む）を全量収録。センサス飲食店を大きく超過。

## 網羅性の最終判定

- **全国スコープではあるが、全国悉皆でも均質でもない。** 比率のばらつき140倍が動かぬ証拠。
- **用途別**：
  - 買い物難民/悉皆分析 → 不可（欠落県で偽の空白を生む）。
  - 相対分布・存在確認 → 網羅の厚い県内でのみ限定的に可。
- **改善余地**：原因A（死にURL 14件）は PR #30 とは別に**ソースURL更新で復旧可能**（愛知・神奈川が大きく改善）。
  原因B・Cは供給側/表記の問題で残る。

## 手法メモ（再現用）
- 正解データ: e-Stat API `getStatsData?statsDataId=0004005687&cdCat01=76`（令和3年、飲食店中分類、全事業所数）。
  市区町村粒度（area 1,966地域）も同じ表で取得可能 → 市区町村別カバー率も算出できる（次段）。
- 許可側: `facilities-all_after.csv` で `business_type LIKE '%飲食店営業%'` を name+address で dedup。
